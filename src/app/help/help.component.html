Manual

ScripDraft is a plain text editor with a robust undo/redo system inspired by Git, but much simpler. (It's OK if you have never heard of Git.)

Overview

Whenever you click "commit", it creates a kind of save point, a version of your text at that point. You can create these save points at any time while drafting your text, except a few special cases added to help avoid mistakes. For example, we do try to keep you from accidentally clicking "commit" twice and saving the same text twice in a row.

After you have created these save points ("commits"), you can review them, clicking the "back" button to see the previous commit or "forward" to view the next commit. Of course, these buttons only apply if there is a previous or a next commit. There would only be a "next" commit if you are currently viewing a past/previous commit.

You will spend most of your time at the head of the commit chain---that is, at the last edit in the chain, possibly with some unsaved changes. In this case, there is no forward commit, so the forward button is disabled.

Primary Editing

When the app first starts, there are no commits.

For various reasons, after you have made some commits, you may want to browse backward through your prior commits and view them, but typically you want to edit only at the tail end of your commit history, thereby creating new information without losing any existing information. To do this, just make sure you click "forward" again until you are on the last commit.

A message below the text editor will tell you something like "At commit x of x" when you are on the most recent saved commit.

Editing from a Past Commit

Much like the traditional undo/redo of a text editing application, if you go back and then edit, then you have lost any ability to go "forward" again. Translated into the terms of this app, if you browse backward to a previous commit and then edit that commit, then you are in the position of making a decision.

If you "commit" your change, then it creates a new next commit, and all of the commits you had previously saved future to the present one are gone.

If you are looking at an interior commit and you make an edit but you decide you don't want to change the history after all, you can still click the "reset" button. Your changes will be lost, but the commit history will be left as it was before you made any changes.

In particular, it is not possible to browse backward to a previous commit, edit it, save the changes, and leave the remaining forward commits as-is. That would have been possible, but we decided from an editing point of view that it makes no sense to allow that.
